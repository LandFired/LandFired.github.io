# 计划规范

1. 制定的计划需包括：需求总结，系统架构，代码模块架构，文件夹结构。

2. 制定计划后，分阶段执行，不要一次给出最终文件

# 代码规范

1. 代码文件必须分成清晰标记的部分。示例如下：

```
/*
文件功能：
一句话说明这个文件干什么
*/

// -------------------- 宏包 --------------------
#include <iostream>
#include <vector>

// -------------------- 配置区 ------------------
/*
所有可修改的参数必须集中在这里
*/
API_KEY = "your_api_key_here"
MODEL_NAME = "gpt-4"

// -------------------- 函数1 --------------------
int add(int a, int b) {
    /*
    每个函数需要说明功能，输入，输出。
    功能：计算两个数的和

    输入：
    a: int
    b: int

    输出：
    int，相加结果
    */
    return a + b;
}

// -------------------- 主函数 --------------------
int main()
{
    return 0;
}
```

2. 代码每个意群都需要一句话注释，说明代码做了什么。示例如下：

```
# 读取用户配置文件
config = load_config()

# 校验配置是否合法
if not validate(config):
    raise Exception("配置错误")
```
